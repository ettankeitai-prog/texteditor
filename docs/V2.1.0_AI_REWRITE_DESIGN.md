# v2.1.0 AI文章整形 設計

> Status: 実装済み

AI文章整形は通常の編集・保存・復元から独立した補助機能です。Codex CLIの未導入、未ログイン、利用枠到達、App Server異常はAI機能だけのエラーとして扱い、文書処理へ波及させません。

## 責務境界

- trusted renderer: CodeMirrorの対象範囲取得、プリセット選択、比較表示、競合検知、CodeMirror transactionによる適用、新規タブ、コピー
- preload: 状態確認・実行・キャンセルだけの限定IPC
- Electron main: sender・引数検証、App Server実行ファイルの解決、ローカルstdioによる遅延起動、認証・モデル・利用枠確認、単一実行、タイムアウト、キャンセル、終了処理
- Codex App Server: `initialize`後に`account/read`、`model/list`、`thread/start`、`turn/start`を処理し、通知ストリームで結果を返す

rendererから実行コマンド、作業ディレクトリ、モデルID、任意のApp Serverメソッドは指定できません。

## 認証と課金方針

ChatGPTアカウントのCodex利用枠だけを利用します。プラン名は固定判定せず、App Serverが返す認証状態・モデル・利用枠を実行時に確認します。APIキー環境変数は子プロセスへ渡さず、APIキー認証が返された場合も実行しません。OpenAI APIキーの入力・保存・自動フォールバックはありません。

## 権限

App Serverは専用の空ディレクトリで起動し、`thread/start`へ`approvalPolicy: "never"`と`sandbox: "read-only"`を固定します。文章整形プロンプトでもファイル、シェル、外部ツールの使用を禁止し、App Serverからのhost action／approval requestは拒否します。

## 本文の取り扱い

本文と生成結果は整形リクエストと比較UIのためだけにメモリ上で扱い、診断ログへ記録しません。診断には実行ファイル解決元、CLI version、spawn／終了状態、request／response ID、JSON-RPC error、タイムアウト種別を記録します。stderrは起動失敗の切り分け用にinitialize応答までの先頭4 KiBだけを共通サニタイズ後に記録します。認証トークン、セッション情報、プロンプト全文は保存しません。

## 競合検知

実行開始時のタブID、小タブID、pane、from/to、対象文字列を保持します。結果適用時に同じCodeMirror documentの同じ範囲が元文字列と一致する場合だけ置換します。不一致時は直接置換を無効にし、新規タブまたはコピーだけを許可します。

## プリセット

- 軽く整える: 誤字脱字、助詞、句読点、表記揺れ、明らかな重複、文法を小さく修正
- 正文化: 意味を変えず、断片的なメモを第三者が読める自然な文章へ整理
- レビュー整形: 元の評価と批判の強さを保ち、Web小説の感想メモを落ち着いたレビューへ整理

すべてのプリセットは共通制約と分離され、本文内の命令文は指示ではなく編集対象として扱います。出力は自動適用しません。
